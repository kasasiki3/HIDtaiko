# HIDtaiko シリアルプロトコル仕様

バージョン: v1.2（ステートレス方式）

---

## 通信方式

- USB CDC シリアル（baud rate: 9600）
- 改行コード: `\n`
- 全コマンドはテキスト形式
- デバイスは全コマンドに対して必ず `OK\n` で応答する

---

## コマンド一覧

### 感度送信 `SENS`

```
JS → device : SENS:0=<v>:1=<v>:2=<v>:3=<v>:4=<v>:5=<v>:6=<v>:7=<v>:8=<v>\n
device → JS : OK\n
```

インデックス対応:

| index | 意味 |
|-------|------|
| 0 | 左縁 感度 |
| 1 | 左面 感度 |
| 2 | 右面 感度 |
| 3 | 右縁 感度 |
| 4 | 面入力後の次の入力受付時間 (B) |
| 5 | 縁の入力受付時間 (C) |
| 6 | 面入力時に縁を無視する時間 (D) |
| 7 | シミュレーター用入力制限 (H) |
| 8 | 単体入力受付時間 (A) |

例:
```
SENS:0=80:1=160:2=160:3=80:4=30:5=12:6=35:7=0:8=0
OK
```

---

### 感度要求 `GETS`

```
JS → device : GETS\n
device → JS : 0:<v>\n
              1:<v>\n
              ...
              8:<v>\n
              OK\n
```

例:
```
GETS
0:80
1:160
2:160
3:80
4:30
5:12
6:35
7:0
8:0
OK
```

---

### キー配置送信 `KEY`

```
JS → device : KEY:0=<code>:1=<code>:2=<code>:3=<code>\n
device → JS : OK\n
```

インデックス対応:

| index | 意味 |
|-------|------|
| 0 | 左縁 キー |
| 1 | 左面 キー |
| 2 | 右面 キー |
| 3 | 右縁 キー |

値は HID キーコード（例: D=0x07, F=0x09, J=0x0D, K=0x0E）

例:
```
KEY:0=7:1=9:2=13:3=14
OK
```

---

### キー配置要求 `GETK`

```
JS → device : GETK\n
device → JS : 0:<code>\n
              1:<code>\n
              2:<code>\n
              3:<code>\n
              OK\n
```

例:
```
GETK
0:7
1:9
2:13
3:14
OK
```

---

### フラッシュ保存 `SAVE`

```
JS → device : SAVE\n
device → JS : OK\n
```

kando[0..8] と key_map[0..3] を Flash に書き込む。

---

### フラッシュ読み込み `LOAD`

```
JS → device : LOAD\n
device → JS : OK\n
```

Flash から kando[0..8] と key_map[0..3] を読み込む。

---

## フラッシュ格納フォーマット

オフセット `0x1F0000` に `uint16_t` 14個分を書き込む。

| buf16 index | 内容 |
|-------------|------|
| 0〜8 | kando[0]〜kando[8] |
| 9 | reserved |
| 10〜13 | key_map[0]〜key_map[3] |

key_map の有効範囲チェック: `4 <= v <= 255`（範囲外はデフォルト値を維持）

---

## JS 側の受信方式

`readLine()` で1行ずつ読み出す。内部的に `lineBuffer` に USB CDC パケットを蓄積し、`\n` 単位で切り出す。

要求コマンド（GETS/GETK）は `"OK"` が来るまでループで読み続ける方式のため、行数を事前に知る必要がない。
